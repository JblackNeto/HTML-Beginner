What are HTML tags?

(content table)

Tags are meant to inform the browser the website's structure. In order words: Whenever a HTML code is written, the tags are understood by the browser,then creating its structure and web design.

How browsers can read an HTML tags?

(content table)

Servers read the HTML code from top to bottom to understand and render its content correctly. You can add as many tags you want. However, there somo essential rules you must follow.

An HTML tag must have:

  1. Opening - By using the < > symbol;
  2. Content - String sentences to show the object in the page;
  3. Closure - By ending with < />

Text Tags

(content table)

Here it starts to get interesting, mainly because we are applying style into you page.

  • < p> < /p > – The paragraph tag is one the most important, simply because you use everytime you start a new text;
  • < span > < /span > – Similar to the paragraph, whenever we use < span >, it roles is for small pieces of information, such as poll's legend, images' legend and so on;
  • < b > < /b > – It changes your text style from regular into bold;
  • < strong > < /strong > - It is used to highlight an important text. However, its style is the same as bold.
  • < i > < /i > – Applies the italic style into your text.
  • < mark > < /mark > - It is used to highlight or emphasize text that is of special interest or relevance within a document.
  • < small > < /small > - It is used to display text in a smaller font size than the surrounding text.
  • < hr > - It defines a thematic break, such as an horizontal rule that helps visualize the text break. An interesting fact about < hr > is, it does not eng a tag.
  • < sub > < /sub > - It defines the subscript text. This kind of text is referred to the one which goes below the line, such as H2O.
  • < sup > < /sup > - Similar to the tag sub. < sup > stands for superscript text, which goes above the line. A famous example is the function E=mc2.
  • < del > < /del > - Defines the text that has been deleted from a document. Browsers will usually strike a line through deleted text

List Tags

(content table)

In this case, there two possibilities we can use.

  • Ordered list – Which can be embeded by adding < ol > < /ol > tags at the beginning of the line.
  • Unordered list – Insert < ul > < /ul >. These are the one with bullet points (black dots). Each list item starts with the < li > tag.
  • Description list – The < dl > tag defines the description list, the < dt > tag defines the term (name), and the < dd > tag describes each term.
